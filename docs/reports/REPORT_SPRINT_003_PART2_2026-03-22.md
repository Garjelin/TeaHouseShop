- Вместо одного потока «все товары» используется **постраничная загрузка** (размер страницы: 4 позиции).
- Состояние успеха расширено: категории, выбранная категория, строка поиска, флаги `isLoadingMore` и `hasMore`.

## Известные ограничения и риски

1. **Порядок относительно mock-инициализации:** `MockDataInitializer` и первая загрузка каталога выполняются асинхронно; теоретически возможен краткий пустой первый кадр до появления данных (как и при потоке из Room до вставки). При необходимости следующим шагом можно синхронизировать старт каталога с завершением инициализации БД.
2. **Поиск без debounce:** при очень больших каталогах имеет смысл добавить задержку ввода; для текущего объёма данных приемлемо.
3. **`searchProductsWithFilters` в репозитории:** по-прежнему заглушка для спринта 7; экран поиска приложения при необходимости позже переводится на общие use cases.
