## 3. Детальный план работ

### Приоритет 1 (Критично для MVP)

#### A. Автономная архитектура с подготовкой к API
**Время:** 1-2 недели

**Текущий подход:** Приложение работает полностью автономно на основе Room Database

1. **Архитектура данных**
   - **Single Source of Truth:** Room Database
   - Mock-данные инициализируются при первом запуске
   - Возможность легко добавить синхронизацию с API в будущем

2. **Подготовка к будущему API (опционально)**
   ```
   // Интерфейсы уже готовы для будущей интеграции:
   POST   /auth/register
   POST   /auth/login
   GET    /products
   GET    /products/:id
   GET    /products/search?q=...&category=...&minPrice=...
   POST   /cart
   POST   /orders
   GET    /orders
   POST   /favourites
   ```

3. **Локальная аутентификация**
   - Хранение данных пользователя в Room
   - Session управление через DataStore Preferences
   - Простая валидация (без JWT пока)

#### B. Data Layer завершение
**Время:** 1 неделя

1. **Маппинг Entity ↔ Domain**
   ```kotlin
   // ProductMapper.kt
   fun ProductEntity.toDomain(): Product
   fun Product.toEntity(): ProductEntity
   ```

2. **Repository pattern (автономный режим)**
   ```kotlin
   class ProductRepositoryImpl(
       private val local: ProductLocalSource
   ) : ProductRepository {
       override fun getProducts(): Flow<Resource<List<Product>>> = 
           flow {
               emit(Resource.Loading)
               
               try {
                   // Инициализация mock-данных при первом запуске
                   local.ensureMockDataInitialized()
                   
                   // Получаем данные из Room
                   local.getProducts().collect { products ->
                       emit(Resource.Success(products))
                   }
               } catch (e: Exception) {
                   emit(Resource.Error(e.toDataError()))
               }
           }
   }
   
   // Для будущей интеграции с API:
   // class ProductRepositoryImpl(
   //     private val remote: ProductRemoteSource, // добавить когда будет API
   //     private val local: ProductLocalSource
   // )
   ```

3. **Offline-first подход**
   - Single Source of Truth = Room Database
   - Mock-данные как постоянный источник
   - Готовность к добавлению синхронизации в будущем

#### C. Use Cases реализация
**Время:** ongoing с каждой фичей

**Структура Use Case:**
```kotlin
class GetProductsUseCase(
    private val repository: ProductRepository
) {
    operator fun invoke(forceRefresh: Boolean = false): Flow<Resource<List<Product>>> =
        repository.getProducts(forceRefresh)
}

class AddToCartUseCase(
    private val repository: CartRepository
) {
    suspend operator fun invoke(product: Product, quantity: Int): Resource<Unit> {
        // Валидация
        if (quantity <= 0) {
            return Resource.Error(ValidationError.InvalidQuantity)
        }
        
        // Бизнес логика
        return repository.addToCart(product, quantity)
    }
}
```

)

class ProductListViewModel(
    private val getProductsUseCase: GetProductsUseCase,
    private val getCategoriesUseCase: GetCategoriesUseCase
) : ViewModel() {
    
    private val _uiState = MutableStateFlow(ProductListUiState())
    val uiState: StateFlow<ProductListUiState> = _uiState.asStateFlow()
    
    init {
        loadProducts()
    }
    
    fun loadProducts(forceRefresh: Boolean = false) {
        viewModelScope.launch {
            getProductsUseCase(forceRefresh).collect { result ->
                when (result) {
                    is Resource.Loading -> {
                        _uiState.update { it.copy(isLoading = true, error = null) }
                    }
                    is Resource.Success -> {
                        _uiState.update { 
                            it.copy(
                                products = result.data,
                                isLoading = false,
                                error = null
                            ) 
                        }
                    }
                    is Resource.Error -> {
                        _uiState.update { 
                            it.copy(
                                isLoading = false,
                                error = result.message
                            ) 
                        }
                    }
                }
            }
        }
    }
    
    fun onCategorySelected(category: String?) {
        _uiState.update { it.copy(selectedCategory = category) }
        loadProducts(forceRefresh = true)
    }
}

## 5. Технический долг

### Текущие проблемы

1. **Room Database не используется**
   - **Проблема:** Данные не кешируются локально
   - **Решение:** Реализовать маппинг и интеграцию с Repository
   - **Приоритет:** Высокий
   - **Время:** 2-3 дня

2. **Mock данные вместо постоянного хранилища**
   - **Проблема:** Данные пересоздаются при каждом запуске
   - **Решение:** Хранить mock-данные в Room DB, инициализировать один раз
   - **Приоритет:** Высокий
   - **Время:** 2-3 дня

3. **Отсутствие Use Cases**
   - **Проблема:** Бизнес-логика смешана с UI
   - **Решение:** Создать Use Cases для всех операций
   - **Приоритет:** Высокий
   - **Время:** ongoing

4. **Неполная реализация экранов**
   - **Проблема:** Многие экраны - пустые заглушки
   - **Решение:** Постепенная реализация по спринтам
   - **Приоритет:** Средний
   - **Время:** ongoing

5. **Нет обработки ошибок**
   - **Проблема:** Приложение может крашиться при сетевых ошибках
   - **Решение:** Добавить try-catch, Resource wrapper, Error UI
   - **Приоритет:** Высокий
   - **Время:** 2-3 дня

6. **Отсутствие тестов**
   - **Проблема:** Нет уверенности в работоспособности кода
   - **Решение:** Писать тесты параллельно с разработкой
   - **Приоритет:** Средний
   - **Время:** ongoing
